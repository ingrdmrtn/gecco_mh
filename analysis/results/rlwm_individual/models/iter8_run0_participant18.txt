Below are three standalone cognitive models that follow the provided template structure. Each returns the negative log-likelihood of the observed choices. They differ in how the WM policy is constructed, how WM is updated, and how WM vs RL arbitration depends on surprise/recency/thresholds and set size.

Note: Assume numpy is already imported as np.

def cognitive_model1(states, actions, rewards, blocks, set_sizes, model_parameters):
    """
    Model 1: Surprise-penalized WM arbitration with global WM decay
    - RL: Rescorla-Wagner with learning rate lr and softmax inverse temperature softmax_beta.
    - WM policy: Deterministic softmax over WM table W_s with high inverse temperature.
    - Arbitration: Mixture of WM and RL where effective WM weight is downregulated by surprise
      (prediction error magnitude from RL) and by set-size cost.
        wm_weight_eff = sigmoid(logit(wm_weight) - pe_sensitivity*|r - Q(s,a)| - size_cost*(nS-3))
    - WM updating: Global decay toward uniform each trial (wm_decay). Rewarded trials consolidate
      the correct association toward a one-hot vector with strength proportional to wm_weight
      (the same parameter used as baseline mixture weight, linking reliance and consolidation).
    - Set-size effect: Larger set size reduces WM contribution via size_cost.

    Parameters (model_parameters):
    - lr: RL learning rate (0..1).
    - wm_weight: Baseline arbitration weight and WM consolidation strength (0..1).
    - softmax_beta: RL inverse temperature; internally scaled by 10.
    - wm_decay: Global decay of WM toward uniform per trial (0..1).
    - pe_sensitivity: How strongly surprise (|r - Q|) suppresses WM reliance (>0).
    - size_cost: Linear penalty of WM reliance with set size (>0).

    Returns:
    - Negative log-likelihood of observed choices.
    """
    lr, wm_weight, softmax_beta, wm_decay, pe_sensitivity, size_cost = model_parameters

    softmax_beta *= 10.0
    softmax_beta_wm = 50.0
    blocks_log_p = 0.0
    for b in np.unique(blocks):

        block_actions = actions[blocks == b]
        block_rewards = rewards[blocks == b]
        block_states = states[blocks == b]
        block_set_sizes = set_sizes[blocks == b]

        nA = 3
        nS = int(block_set_sizes[0])

        q = (1.0 / nA) * np.ones((nS, nA))
        w = (1.0 / nA) * np.ones((nS, nA))
        w_0 = (1.0 / nA) * np.ones((nS, nA))

        log_p = 0.0
        for t in range(len(block_states)):

            a = int(block_actions[t])
            s = int(block_states[t])
            r = float(block_rewards[t])

            Q_s = q[s, :]
            W_s = w[s, :]

            # RL policy
            denom_rl = np.sum(np.exp(softmax_beta * (Q_s - Q_s[a])))
            p_rl = 1.0 / max(denom_rl, 1e-12)

            # WM policy: sharp softmax over WM table
            denom_wm = np.sum(np.exp(softmax_beta_wm * (W_s - W_s[a])))
            p_wm = 1.0 / max(denom_wm, 1e-12)

            # Arbitration: baseline wm_weight, penalized by surprise and set size
            pe = abs(r - Q_s[a])
            logit_base = np.log(np.clip(wm_weight, 1e-6, 1 - 1e-6)) - np.log(np.clip(1 - wm_weight, 1e-6, 1 - 1e-6))
            penalty = pe_sensitivity * pe + size_cost * max(0, nS - 3)
            wm_weight_eff = 1.0 / (1.0 + np.exp(-(logit_base - penalty)))

            p_total = wm_weight_eff * p_wm + (1.0 - wm_weight_eff) * p_rl
            p_total = max(p_total, 1e-12)
            log_p += np.log(p_total)

            # RL update
            delta = r - Q_s[a]
            q[s, a] += lr * delta

            # WM decay toward uniform
            w[s, :] = (1.0 - wm_decay) * w[s, :] + wm_decay * w_0[s, :]

            # WM consolidation on reward: pull toward one-hot
            if r > 0.0:
                one_hot = np.zeros(nA)
                one_hot[a] = 1.0
                w[s, :] = (1.0 - wm_weight) * w[s, :] + wm_weight * one_hot

        blocks_log_p += log_p

    return -blocks_log_p


def cognitive_model2(states, actions, rewards, blocks, set_sizes, model_parameters):
    """
    Model 2: Recency-gated WM with set-size modulation and choice stickiness in arbitration
    - RL: Rescorla-Wagner with learning rate lr and softmax inverse temperature softmax_beta.
    - WM policy: Sharp softmax over WM table W_s.
    - Arbitration: WM weight increases with state recency (time since last visit is shorter),
      decreases with set size, and is biased by a stickiness kernel favoring the last action taken
      in that state. The effective mixture is:
        wm_weight_eff = sigmoid(logit(wm_weight)
                                + recency_sensitivity * recency_score[s]
                                - size_sensitivity * (nS - 3)
                                + stickiness * I[a == last_action_in_state[s]])
      where recency_score decays with the gap since last visit.
    - WM updating: Rewarded trials strengthen WM mapping toward chosen action proportional to wm_weight.
      Unrewarded trials mildly forget toward uniform via the same consolidation term.
    - Set-size effect: Larger sets reduce WM reliance via size_sensitivity.

    Parameters (model_parameters):
    - lr: RL learning rate (0..1).
    - wm_weight: Baseline WM weight and consolidation strength (0..1).
    - softmax_beta: RL inverse temperature; internally scaled by 10.
    - recency_sensitivity: Weight for recency gating (>0).
    - size_sensitivity: Linear penalty of WM reliance with set size (>0).

    Returns:
    - Negative log-likelihood of observed choices.
    """
    lr, wm_weight, softmax_beta, recency_sensitivity, size_sensitivity = model_parameters

    softmax_beta *= 10.0
    softmax_beta_wm = 50.0
    blocks_log_p = 0.0

    for b in np.unique(blocks):

        block_actions = actions[blocks == b]
        block_rewards = rewards[blocks == b]
        block_states = states[blocks == b]
        block_set_sizes = set_sizes[blocks == b]

        nA = 3
        nS = int(block_set_sizes[0])

        q = (1.0 / nA) * np.ones((nS, nA))
        w = (1.0 / nA) * np.ones((nS, nA))
        w_0 = (1.0 / nA) * np.ones((nS, nA))

        # Track recency and last action per state
        last_visit_t = -1 * np.ones(nS, dtype=float)
        last_action = -1 * np.ones(nS, dtype=int)

        log_p = 0.0
        for t in range(len(block_states)):

            a = int(block_actions[t])
            s = int(block_states[t])
            r = float(block_rewards[t])

            Q_s = q[s, :]
            W_s = w[s, :]

            # RL policy
            denom_rl = np.sum(np.exp(softmax_beta * (Q_s - Q_s[a])))
            p_rl = 1.0 / max(denom_rl, 1e-12)

            # WM policy
            denom_wm = np.sum(np.exp(softmax_beta_wm * (W_s - W_s[a])))
            p_wm = 1.0 / max(denom_wm, 1e-12)

            # Recency score: 1 if immediately revisited, decays as elapsed grows
            if last_visit_t[s] < 0:
                recency_score = 0.0
            else:
                gap = t - last_visit_t[s]
                recency_score = 1.0 / (1.0 + gap)  # simple hyperbolic decay

            # Stickiness bonus if repeating last action in this state
            sticky = 1.0 if (last_action[s] == a and last_action[s] >= 0) else 0.0

            # Arbitration weight: baseline plus recency minus set-size plus stickiness
            logit_base = np.log(np.clip(wm_weight, 1e-6, 1 - 1e-6)) - np.log(np.clip(1 - wm_weight, 1e-6, 1 - 1e-6))
            size_term = size_sensitivity * max(0, nS - 3)
            wm_weight_eff = 1.0 / (1.0 + np.exp(-(logit_base + recency_sensitivity * recency_score - size_term + sticky)))

            p_total = wm_weight_eff * p_wm + (1.0 - wm_weight_eff) * p_rl
            p_total = max(p_total, 1e-12)
            log_p += np.log(p_total)

            # RL update
            delta = r - Q_s[a]
            q[s, a] += lr * delta

            # WM update: reward strengthens mapping, no-reward forgets toward uniform
            one_hot = np.zeros(nA)
            one_hot[a] = 1.0
            if r > 0.0:
                w[s, :] = (1.0 - wm_weight) * w[s, :] + wm_weight * one_hot
            else:
                w[s, :] = (1.0 - wm_weight) * w[s, :] + wm_weight * w_0[s, :]

            # Update bookkeeping
            last_visit_t[s] = t
            last_action[s] = a

        blocks_log_p += log_p

    return -blocks_log_p


def cognitive_model3(states, actions, rewards, blocks, set_sizes, model_parameters):
    """
    Model 3: Thresholded WM access with set-sizeâ€“dependent criterion and sharp arbitration
    - RL: Rescorla-Wagner with learning rate lr and softmax inverse temperature softmax_beta.
    - WM policy: Sharp softmax over WM table W_s.
    - Arbitration: Gate uses WM if the WM "confidence" (max(W_s) - mean(W_s)) exceeds a threshold.
      The threshold rises with set size:
         theta(nS) = theta_base + theta_size_slope * (nS - 3)
      The mixture weight is a steep sigmoid around the margin:
         wm_weight_eff = sigmoid(gamma * (confidence - theta(nS) + logit(wm_weight)))
      so wm_weight acts as a prior bias toward WM when near threshold.
    - WM updating: Decay toward uniform each trial; on reward, consolidate toward chosen action
      proportional to wm_weight (linking reliance and consolidation).

    Parameters (model_parameters):
    - lr: RL learning rate (0..1).
    - wm_weight: Baseline prior bias toward WM and WM consolidation strength (0..1).
    - softmax_beta: RL inverse temperature; internally scaled by 10.
    - theta_base: Base WM confidence threshold at set size 3.
    - theta_size_slope: Increase of the threshold per additional item (>0).
    - gamma: Steepness of the gating nonlinearity (>0).

    Returns:
    - Negative log-likelihood of observed choices.
    """
    lr, wm_weight, softmax_beta, theta_base, theta_size_slope, gamma = model_parameters

    softmax_beta *= 10.0
    softmax_beta_wm = 50.0
    wm_decay = 0.1  # light default decay; reliance still controlled by wm_weight and threshold

    blocks_log_p = 0.0
    for b in np.unique(blocks):

        block_actions = actions[blocks == b]
        block_rewards = rewards[blocks == b]
        block_states = states[blocks == b]
        block_set_sizes = set_sizes[blocks == b]

        nA = 3
        nS = int(block_set_sizes[0])

        q = (1.0 / nA) * np.ones((nS, nA))
        w = (1.0 / nA) * np.ones((nS, nA))
        w_0 = (1.0 / nA) * np.ones((nS, nA))

        threshold = theta_base + theta_size_slope * max(0, nS - 3)

        log_p = 0.0
        for t in range(len(block_states)):

            a = int(block_actions[t])
            s = int(block_states[t])
            r = float(block_rewards[t])

            Q_s = q[s, :]
            W_s = w[s, :]

            # RL policy
            denom_rl = np.sum(np.exp(softmax_beta * (Q_s - Q_s[a])))
            p_rl = 1.0 / max(denom_rl, 1e-12)

            # WM policy
            denom_wm = np.sum(np.exp(softmax_beta_wm * (W_s - W_s[a])))
            p_wm = 1.0 / max(denom_wm, 1e-12)

            # Confidence-based gating with set-size threshold and prior bias (wm_weight)
            confidence = float(np.max(W_s) - np.mean(W_s))
            prior_logit = np.log(np.clip(wm_weight, 1e-6, 1 - 1e-6)) - np.log(np.clip(1 - wm_weight, 1e-6, 1 - 1e-6))
            wm_weight_eff = 1.0 / (1.0 + np.exp(-gamma * (confidence - threshold) - prior_logit))

            p_total = wm_weight_eff * p_wm + (1.0 - wm_weight_eff) * p_rl
            p_total = max(p_total, 1e-12)
            log_p += np.log(p_total)

            # RL update
            delta = r - Q_s[a]
            q[s, a] += lr * delta

            # WM decay toward uniform and reward-based consolidation
            w[s, :] = (1.0 - wm_decay) * w[s, :] + wm_decay * w_0[s, :]
            if r > 0.0:
                one_hot = np.zeros(nA)
                one_hot[a] = 1.0
                w[s, :] = (1.0 - wm_weight) * w[s, :] + wm_weight * one_hot

        blocks_log_p += log_p

    return -blocks_log_p

Modeling notes on set-size effects:
- Model 1 reduces WM reliance linearly with set size via size_cost, and also via surprise (large PE -> less WM).
- Model 2 reduces WM reliance with set size via size_sensitivity and increases reliance when the state is revisited soon (recency_sensitivity).
- Model 3 raises the WM confidence threshold with set size via theta_size_slope, making WM less likely to dominate in larger sets.
