Here are three new cognitive models for the two-step task.

### Model 1: Dual-Alpha Hybrid Model
This model hypothesizes that the participant learns at different rates for the two stages of the task. While the "Hybrid" architecture (mixing Model-Free and Model-Based) is standard, this version decouples the learning rate for the high-level spaceship choice (`alpha_1`) from the low-level alien choice (`alpha_2`). This allows the model to capture scenarios where a participant might be quick to learn which alien is currently rich (high `alpha_2`) but slow to update their habitual preference for spaceships (low `alpha_1`), or vice versa.

```python
def cognitive_model1(action_1, state, action_2, reward, model_parameters):
    """
    Hybrid Model with Dual Learning Rates.
    
    This model combines Model-Free (MF) and Model-Based (MB) strategies.
    It uses separate learning rates for the Stage 1 MF update (associating 
    spaceships directly with rewards via TD(1)) and the Stage 2 value update 
    (learning alien values). This captures potential differences in learning 
    speed between the abstract high-level choice and the concrete outcome.
    
    Parameters:
    - alpha_1: [0, 1] Learning rate for Stage 1 MF values.
    - alpha_2: [0, 1] Learning rate for Stage 2 values.
    - beta_1: [0, 10] Inverse temperature for Stage 1 choices.
    - beta_2: [0, 10] Inverse temperature for Stage 2 choices.
    - w: [0, 1] Weighting parameter (0 = Pure MF, 1 = Pure MB).
    - stickiness: [0, 5] Choice stickiness for Stage 1.
    """
    alpha_1, alpha_2, beta_1, beta_2, w, stickiness = model_parameters
    n_trials = len(action_1)
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Q-values
    q_stage1_mf = np.zeros(2) # MF value for stage 1 (TD(1))
    q_stage2_mf = np.zeros((2, 2)) # Values for aliens (State x Action)
    
    prev_action_1 = -1

    for trial in range(n_trials):
        # Handle missing/invalid trials
        if action_1[trial] == -1:
            p_choice_1[trial] = 1.0
            p_choice_2[trial] = 1.0
            continue

        # --- Stage 1 Policy ---
        # Model-Based Value: Expected value of next stage based on transition matrix
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Hybrid Value: Weighted sum of MB and MF values
        q_hybrid = w * q_stage1_mb + (1 - w) * q_stage1_mf
        
        # Stickiness (repetition bias)
        stick_vec = np.zeros(2)
        if prev_action_1 != -1:
            stick_vec[prev_action_1] = stickiness
            
        # Softmax Choice 1
        exp_q1 = np.exp(beta_1 * (q_hybrid + stick_vec))
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[action_1[trial]]
        
        # --- Stage 2 Policy ---
        state_idx = state[trial]
        exp_q2 = np.exp(beta_2 * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[action_2[trial]]
        
        # --- Updates ---
        # Stage 2 Update (Standard Q-learning)
        delta_stage2 = reward[trial] - q_stage2_mf[state_idx, action_2[trial]]
        q_stage2_mf[state_idx, action_2[trial]] += alpha_2 * delta_stage2
        
        # Stage 1 MF Update (TD(1))
        # Updates Stage 1 value directly based on the final reward
        delta_stage1_mf = reward[trial] - q_stage1_mf[action_1[trial]]
        q_stage1_mf[action_1[trial]] += alpha_1 * delta_stage1_mf
        
        prev_action_1 = action_1[trial]

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: Model-Based RL with Choice Kernel (Habit)
The participant data shows long "blocks" of repeating the same spaceship choice (e.g., trials 15-51, 62-87). A simple 1-back stickiness parameter might not capture this momentum well. This model replaces simple stickiness with a "Choice Kernel"â€”a decaying memory of past choices. This represents a "habit" strength that builds up with repetition and decays slowly, competing with or aiding the Goal-Directed (Model-Based) system.

```python
def cognitive_model2(action_1, state, action_2, reward, model_parameters):
    """
    Model-Based RL with Choice Kernel (Habit).
    
    This model assumes the participant uses a Model-Based strategy for value 
    estimation, but their Stage 1 choices are heavily influenced by a "Choice Kernel" 
    (habit). Unlike simple stickiness, the choice kernel accumulates over time 
    and decays, capturing the momentum seen in the participant's long streaks 
    of repeating the same spaceship.
    
    Parameters:
    - alpha: [0, 1] Learning rate for Stage 2 alien values.
    - beta_mb: [0, 10] Weight (inverse temp) for Model-Based values in Stage 1.
    - beta_habit: [0, 10] Weight for the Choice Kernel (Habit) in Stage 1.
    - alpha_k: [0, 1] Decay rate for the Choice Kernel (higher = faster decay).
    - beta_2: [0, 10] Inverse temperature for Stage 2.
    """
    alpha, beta_mb, beta_habit, alpha_k, beta_2 = model_parameters
    n_trials = len(action_1)
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)

    q_stage2_mf = np.zeros((2, 2))
    choice_kernel = np.zeros(2) # Tracks habit strength for spaceship 0 and 1

    for trial in range(n_trials):
        if action_1[trial] == -1:
            p_choice_1[trial] = 1.0
            p_choice_2[trial] = 1.0
            continue

        # --- Stage 1 Policy ---
        # MB Value
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Combined utility: Goal-Directed Value + Habitual Kernel
        utilities = beta_mb * q_stage1_mb + beta_habit * choice_kernel
        
        exp_u1 = np.exp(utilities)
        probs_1 = exp_u1 / np.sum(exp_u1)
        p_choice_1[trial] = probs_1[action_1[trial]]
        
        # --- Stage 2 Policy ---
        state_idx = state[trial]
        exp_q2 = np.exp(beta_2 * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[action_2[trial]]
        
        # --- Updates ---
        # Value Update
        delta = reward[trial] - q_stage2_mf[state_idx, action_2[trial]]
        q_stage2_mf[state_idx, action_2[trial]] += alpha * delta
        
        # Choice Kernel Update
        # K[a] decays towards 0, but gets a boost if 'a' was chosen.
        mask = np.zeros(2)
        mask[action_1[trial]] = 1
        choice_kernel = (1 - alpha_k) * choice_kernel + alpha_k * mask

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: Model-Based RL with Counterfactual Updating
This model tests the hypothesis that the participant engages in "fictitious play" or counterfactual reasoning. When they choose an alien and receive a reward (0 or 1), they not only update the chosen alien but also update the *unchosen* alien as if it had yielded the opposite result (1 - reward). This reflects a belief structure where the participant assumes the two aliens on a planet are anti-correlated (i.e., if one has gold, the other doesn't).

```python
def cognitive_model3(action_1, state, action_2, reward, model_parameters):
    """
    Model-Based RL with Counterfactual Updating.
    
    This model assumes the participant is Model-Based but engages in 
    "fictitious play" at Stage 2. When they receive a reward from one alien, 
    they update the value of the *unchosen* alien as if they had received 
    the opposite outcome (1 - reward). This reflects a belief that the aliens 
    are anti-correlated (e.g. "If this one gave me nothing, the other one 
    must have the coin").
    
    Parameters:
    - alpha: [0, 1] Learning rate for the chosen option.
    - alpha_cf: [0, 1] Learning rate for the unchosen option (counterfactual).
    - beta_1: [0, 10] Inverse temperature for Stage 1.
    - beta_2: [0, 10] Inverse temperature for Stage 2.
    - stickiness: [0, 5] Simple stickiness for Stage 1.
    """
    alpha, alpha_cf, beta_1, beta_2, stickiness = model_parameters
    n_trials = len(action_1)
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)

    q_stage2_mf = np.zeros((2, 2))
    prev_action_1 = -1

    for trial in range(n_trials):
        if action_1[trial] == -1:
            p_choice_1[trial] = 1.0
            p_choice_2[trial] = 1.0
            continue

        # --- Stage 1 Policy (Model-Based) ---
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        stick_vec = np.zeros(2)
        if prev_action_1 != -1:
            stick_vec[prev_action_1] = stickiness

        exp_q1 = np.exp(beta_1 * (q_stage1_mb + stick_vec))
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[action_1[trial]]
        
        # --- Stage 2 Policy ---
        state_idx = state[trial]
        exp_q2 = np.exp(beta_2 * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[action_2[trial]]
        
        # --- Updates ---
        chosen_a2 = action_2[trial]
        unchosen_a2 = 1 - chosen_a2
        r = reward[trial]
        
        # Update Chosen (Standard)
        q_stage2_mf[state_idx, chosen_a2] += alpha * (r - q_stage2_mf[state_idx, chosen_a2])
        
        # Update Unchosen (Counterfactual)
        # Assume unchosen would have yielded (1 - r)
        cf_reward = 1 - r
        q_stage2_mf[state_idx, unchosen_a2] += alpha_cf * (cf_reward - q_stage2_mf[state_idx, unchosen_a2])
        
        prev_action_1 = action_1[trial]

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
