Here are three cognitive models designed to explain the participant's behavior, incorporating their low OCI-R score (0.2375) into the decision-making mechanics.

### Model 1: Hybrid Learner with OCI-Modulated Model-Based Weight
This model assumes that the balance between Model-Based (planning) and Model-Free (habitual) control is influenced by the participant's obsessive-compulsive traits. Given the low OCI score (indicating low compulsivity), this participant might rely more heavily on goal-directed (Model-Based) strategies compared to someone with high compulsivity (who might be more habitual). The parameter `w` (mixing weight) is modulated by the OCI score.

```python
def cognitive_model1(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid Model-Based/Model-Free learner where the weighting (w) is modulated by OCI.
    
    The mixing weight 'w' determines the balance between Model-Based (MB) and Model-Free (MF) control.
    w = 1 implies pure MB, w = 0 implies pure MF.
    Here, the effective w is influenced by the base_w parameter and the inverse of the OCI score,
    reflecting the hypothesis that lower compulsivity (low OCI) allows for more flexible, model-based planning.

    Parameters:
    - learning_rate: [0, 1] Learning rate for Q-value updates.
    - beta: [0, 10] Inverse temperature for softmax choice.
    - base_w: [0, 1] Base weight for model-based control.
    - oci_sensitivity: [0, 1] How strongly OCI reduces model-based control.
    """
    learning_rate, beta, base_w, oci_sensitivity = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0] # 0.2375 for this participant
  
    # Fixed transition matrix: A->X (0.7), U->Y (0.7)
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Q-values
    q_stage1_mf = np.zeros(2)      # MF values for Stage 1 (Spaceships)
    q_stage2_mf = np.zeros((2, 2)) # MF values for Stage 2 (Aliens) - shared for MB calculation

    # Calculate effective mixing weight w. 
    # Logic: Higher OCI (more compulsivity) reduces MB control.
    # We clip to ensure it stays within [0, 1].
    w = np.clip(base_w - (current_oci * oci_sensitivity), 0.0, 1.0)

    for trial in range(n_trials):
        # --- STAGE 1 CHOICE ---
        # Model-Based Value: Transition * Max(Stage2)
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Hybrid Value
        q_net = (w * q_stage1_mb) + ((1 - w) * q_stage1_mf)
        
        # Softmax Policy Stage 1
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        # --- STAGE 2 CHOICE ---
        state_idx = int(state[trial]) # Planet reached (0 or 1)
        
        # Standard Model-Free Q-learning for Stage 2
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- UPDATES ---
        # Prediction errors
        # Note: In standard TD(1) or hybrid models, Stage 1 MF is updated by the value of the state reached (or max of next state)
        # Here we use SARSA-like update or Max-Q update. Using Max-Q of stage 2 for Stage 1 update.
        
        # Update Stage 1 MF
        # We use the value of the chosen stage 2 action to update stage 1
        chosen_s2_val = q_stage2_mf[state_idx, int(action_2[trial])]
        delta_stage1 = chosen_s2_val - q_stage1_mf[int(action_1[trial])]
        q_stage1_mf[int(action_1[trial])] += learning_rate * delta_stage1
        
        # Update Stage 2 MF
        delta_stage2 = reward[trial] - chosen_s2_val
        q_stage2_mf[state_idx, int(action_2[trial])] += learning_rate * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: OCI-Driven Perseveration (Stickiness)
This model posits that OCI scores relate to "stickiness" or choice perseverationâ€”the tendency to repeat the previous action regardless of reward. Even though this participant has a low OCI, the model allows the data to determine if their "low" score translates to low stickiness (flexible switching) or if there is a baseline level of habit.

```python
def cognitive_model2(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Free learner with Choice Perseveration (Stickiness) modulated by OCI.
    
    This model adds a 'stickiness' bonus to the previously chosen action.
    The magnitude of this stickiness is determined by a base parameter plus an OCI-dependent factor.
    
    Parameters:
    - learning_rate: [0, 1] Learning rate.
    - beta: [0, 10] Inverse temperature.
    - stick_base: [0, 5] Base bonus for repeating the last choice.
    - oci_stick_mod: [0, 5] Additional stickiness scaling with OCI score.
    """
    learning_rate, beta, stick_base, oci_stick_mod = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]

    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    q_stage1 = np.zeros(2)
    q_stage2 = np.zeros((2, 2))
    
    # Track last choice (initialize with -1 or None, handle in loop)
    last_action_1 = -1

    # Calculate effective stickiness
    # Low OCI -> Lower effective stickiness (easier to switch strategies)
    eff_stickiness = stick_base + (current_oci * oci_stick_mod)

    for trial in range(n_trials):

        # --- STAGE 1 CHOICE ---
        # Add stickiness bonus to Q-values temporarily for decision making
        q_net_1 = q_stage1.copy()
        if last_action_1 != -1:
            q_net_1[int(last_action_1)] += eff_stickiness
            
        exp_q1 = np.exp(beta * q_net_1)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        # Update tracker
        last_action_1 = action_1[trial]
        state_idx = int(state[trial])

        # --- STAGE 2 CHOICE ---
        # Standard choice at stage 2 (no stickiness modeled here to keep params low)
        exp_q2 = np.exp(beta * q_stage2[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- UPDATES ---
        # TD(0) updates
        
        # Update Stage 1
        # Using the Q-value of the state reached (max Q of next stage)
        max_next_q = np.max(q_stage2[state_idx])
        delta_stage1 = max_next_q - q_stage1[int(action_1[trial])]
        q_stage1[int(action_1[trial])] += learning_rate * delta_stage1
        
        # Update Stage 2
        delta_stage2 = reward[trial] - q_stage2[state_idx, int(action_2[trial])]
        q_stage2[state_idx, int(action_2[trial])] += learning_rate * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: OCI-Modulated Reward Sensitivity (Learning Rate)
This model tests the hypothesis that OCI scores affect how aggressively a participant learns from prediction errors. A low OCI score (low anxiety/compulsivity) might correlate with a more stable, optimal learning rate, whereas high scores might lead to over-correction (high alpha) or under-reaction. Here, the learning rate is a function of the base rate and the OCI score.

```python
def cognitive_model3(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Based learner where the Learning Rate is modulated by OCI.
    
    This model assumes the participant is primarily Model-Based (planning using the transition matrix),
    but their sensitivity to reward prediction errors (learning rate) is influenced by their OCI score.
    
    Parameters:
    - base_lr: [0, 1] Base learning rate.
    - beta: [0, 10] Inverse temperature.
    - oci_lr_mod: [0, 1] Factor by which OCI dampens or boosts the learning rate.
    """
    base_lr, beta, oci_lr_mod = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Only Stage 2 Q-values are needed for a pure Model-Based agent (Stage 1 is calculated)
    q_stage2 = np.zeros((2, 2))

    # Calculate effective learning rate
    # We model it such that higher OCI might increase volatility (higher LR) or decrease it.
    # Here: LR = base_lr * (1 + OCI * mod) clipped to [0,1]
    # This allows the optimizer to find if OCI amplifies or reduces learning speed.
    effective_lr = np.clip(base_lr * (1.0 + (current_oci * oci_lr_mod)), 0.0, 1.0)

    for trial in range(n_trials):

        # --- STAGE 1 CHOICE (Pure Model-Based) ---
        max_q_stage2 = np.max(q_stage2, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        exp_q1 = np.exp(beta * q_stage1_mb)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        state_idx = int(state[trial])

        # --- STAGE 2 CHOICE ---
        exp_q2 = np.exp(beta * q_stage2[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- UPDATES ---
        # Only need to update Stage 2 values based on reward
        delta_stage2 = reward[trial] - q_stage2[state_idx, int(action_2[trial])]
        
        # Use the OCI-modulated effective learning rate
        q_stage2[state_idx, int(action_2[trial])] += effective_lr * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
