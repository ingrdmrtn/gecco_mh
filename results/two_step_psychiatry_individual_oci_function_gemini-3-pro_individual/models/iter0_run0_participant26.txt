Here are three cognitive models that incorporate the OCI-R score into the decision-making process for the two-step task.

### Model 1: OCI-Modulated Model-Based/Model-Free Balance
This model hypothesizes that the balance between Model-Based (planning) and Model-Free (habitual) control is influenced by the OCI score. Specifically, it tests whether higher OCI scores shift reliance towards habitual (Model-Free) control or planning (Model-Based) control. The parameter `w` (mixing weight) is dynamically adjusted by the OCI score.

```python
def cognitive_model1(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model 1: OCI-Modulated MB/MF Balance.
    
    This model assumes the OCI score modulates the mixing weight 'w' between 
    Model-Based (MB) and Model-Free (MF) values.
    
    Parameters:
    - learning_rate: [0, 1] Rate at which Q-values are updated.
    - beta: [0, 10] Inverse temperature for softmax choice.
    - w_base: [0, 1] Baseline mixing weight (0=pure MF, 1=pure MB).
    - oci_sens: [0, 1] Sensitivity of the mixing weight to the OCI score.
    """
    learning_rate, beta, w_base, oci_sens = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
  
    # Define transition matrix (fixed knowledge of task structure)
    # Row 0: Spaceship A -> [Planet X (0), Planet Y (1)]
    # Row 1: Spaceship U -> [Planet X (0), Planet Y (1)]
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Q-values
    q_stage1_mf = np.zeros(2)       # Model-free values for Stage 1 (Spaceships)
    q_stage2_mf = np.zeros((2, 2))  # Model-free values for Stage 2 (Aliens per Planet)

    # Modulate mixing weight w based on OCI
    # We use a sigmoid-like transformation or simple linear scaling clipped to [0,1]
    # Hypothesis: Does OCI increase or decrease MB control?
    # Here: w = w_base + (oci_sens * (current_oci - 0.5))
    # Note: OCI is roughly 0 to 1 based on provided buckets, though raw scores vary.
    # Given the low score (0.1625), this term will likely reduce w if oci_sens is positive.
    w = w_base + oci_sens * (current_oci - 0.3) # Centering around the low/med cutoff
    w = np.clip(w, 0.0, 1.0)

    for trial in range(n_trials):
        
        # --- Stage 1 Choice Policy ---
        # 1. Calculate Model-Based values
        # Max value available at stage 2 for each planet
        max_q_stage2 = np.max(q_stage2_mf, axis=1) 
        # Weighted by transition probabilities
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # 2. Integrate MB and MF values
        q_net = w * q_stage1_mb + (1 - w) * q_stage1_mf
        
        # 3. Softmax
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        # --- Stage 2 Choice Policy ---
        state_idx = int(state[trial]) # Which planet did we land on?
        
        # Standard Softmax on Stage 2 Q-values
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx, :])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- Learning / Updates ---
        
        # Prediction errors
        # Stage 1 MF update (TD(0)) - driven by value of state arrived at
        # Often in 2-step tasks, Stage 1 MF is updated by the value of the *chosen* stage 2 option
        # or the max of the next stage. Here we use the chosen option (SARSA-like).
        delta_stage1 = q_stage2_mf[state_idx, int(action_2[trial])] - q_stage1_mf[int(action_1[trial])]
        q_stage1_mf[int(action_1[trial])] += learning_rate * delta_stage1
        
        # Stage 2 MF update - driven by reward
        delta_stage2 = reward[trial] - q_stage2_mf[state_idx, int(action_2[trial])]
        q_stage2_mf[state_idx, int(action_2[trial])] += learning_rate * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: OCI-Driven Perseveration (Stickiness)
This model hypothesizes that OCI symptoms manifest as "stickiness" or perseverationâ€”a tendency to repeat the previous choice regardless of reward. A participant with OCI traits might feel a compulsion to repeat an action. The `stickiness` parameter is scaled by the OCI score.

```python
def cognitive_model2(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model 2: OCI-Driven Perseveration (Stickiness).
    
    This model assumes OCI modulates choice stickiness (perseveration).
    It uses a purely Model-Based planner for Stage 1 values, but adds a 
    stickiness bonus to the previously chosen option, the magnitude of which 
    depends on the OCI score.
    
    Parameters:
    - learning_rate: [0, 1] Rate at which Q-values are updated.
    - beta: [0, 10] Inverse temperature.
    - stick_base: [0, 5] Baseline tendency to repeat the last choice.
    - oci_stick_mod: [0, 5] Additional stickiness modulated by OCI score.
    """
    learning_rate, beta, stick_base, oci_stick_mod = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # We will track only Stage 2 MF values, assuming Stage 1 is purely MB + Stickiness
    q_stage2_mf = np.zeros((2, 2))
    
    # Track previous choice for stickiness (initialize to -1 so no stickiness on trial 0)
    last_choice_1 = -1

    # Calculate total stickiness bonus
    # If OCI is high, stickiness increases
    total_stickiness = stick_base + (current_oci * oci_stick_mod)

    for trial in range(n_trials):

        # --- Stage 1 Choice Policy ---
        # Pure Model-Based Calculation
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Add Stickiness Bonus
        logits = beta * q_stage1_mb
        if last_choice_1 != -1:
            logits[last_choice_1] += total_stickiness
            
        exp_q1 = np.exp(logits)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        # Update last choice
        last_choice_1 = int(action_1[trial])
        
        state_idx = int(state[trial])

        # --- Stage 2 Choice Policy ---
        # Standard Softmax
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx, :])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- Learning ---
        # Only updating Stage 2 values based on reward
        delta_stage2 = reward[trial] - q_stage2_mf[state_idx, int(action_2[trial])]
        q_stage2_mf[state_idx, int(action_2[trial])] += learning_rate * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: OCI-Modulated Learning Rate Asymmetry
This model suggests that OCI affects how participants learn from positive versus negative outcomes. Specifically, it tests if higher OCI scores lead to hyper-sensitivity to punishment (or lack of reward) or reward. We implement separate learning rates for positive and negative prediction errors, where the balance is shifted by OCI.

```python
def cognitive_model3(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model 3: OCI-Modulated Learning Rate Asymmetry.
    
    This model assumes OCI affects the ratio between learning from positive prediction errors 
    (getting more than expected) vs negative prediction errors (getting less).
    It uses a simple Model-Free temporal difference learning scheme for both stages.
    
    Parameters:
    - lr_base: [0, 1] Base learning rate.
    - beta: [0, 10] Inverse temperature.
    - oci_asymmetry: [0, 1] How much OCI biases learning towards negative errors.
    - eligibility: [0, 1] Eligibility trace (lambda) to update Stage 1 based on Stage 2 outcome.
    """
    lr_base, beta, oci_asymmetry, eligibility = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
  
    # No transition matrix used here; pure Model-Free logic
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))

    # Define modulated learning rates
    # If oci_asymmetry is high and OCI is high, lr_neg becomes larger than lr_pos
    # This simulates anxiety/compulsion driven by fear of negative outcomes
    # Note: We clamp to ensure [0,1]
    
    # Base modification factor
    mod = current_oci * oci_asymmetry
    
    lr_pos = np.clip(lr_base - mod, 0.01, 1.0) # Decrease pos learning with high OCI
    lr_neg = np.clip(lr_base + mod, 0.01, 1.0) # Increase neg learning with high OCI

    for trial in range(n_trials):

        # --- Stage 1 Choice Policy ---
        exp_q1 = np.exp(beta * q_stage1_mf)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        state_idx = int(state[trial])

        # --- Stage 2 Choice Policy ---
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx, :])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
  
        # --- Learning ---
        
        # 1. Stage 1 Update (TD(0))
        # Update Stage 1 based on the value of the state reached (Stage 2 Q-value)
        target_stage1 = q_stage2_mf[state_idx, int(action_2[trial])]
        delta_stage1 = target_stage1 - q_stage1_mf[int(action_1[trial])]
        
        lr_1 = lr_pos if delta_stage1 >= 0 else lr_neg
        q_stage1_mf[int(action_1[trial])] += lr_1 * delta_stage1
        
        # 2. Stage 2 Update
        # Update Stage 2 based on reward
        delta_stage2 = reward[trial] - q_stage2_mf[state_idx, int(action_2[trial])]
        
        lr_2 = lr_pos if delta_stage2 >= 0 else lr_neg
        q_stage2_mf[state_idx, int(action_2[trial])] += lr_2 * delta_stage2
        
        # 3. Eligibility Trace Update (TD(1) / Sarsa-lambda aspect)
        # Allows the Stage 2 reward to directly influence Stage 1 choice
        # This connects the final reward back to the first spaceship choice
        q_stage1_mf[int(action_1[trial])] += lr_1 * eligibility * delta_stage2
        

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
