Here are three new cognitive models exploring different mechanisms for how OCI might influence decision-making in the two-step task. These models move beyond simple modulation of beta or learning rates and investigate the balance between model-based/model-free control and the specific impact of reward valence.

### Model 1: Hybrid Learner with OCI-modulated Model-Based Weight (`w`)
This model hypothesizes that OCI symptoms specifically affect the balance between goal-directed (Model-Based) and habitual (Model-Free) control. High OCI might lead to an over-reliance on habitual systems (lower `w`) or, conversely, rigid goal-directed control. This model allows the mixing parameter `w` to vary linearly with OCI score.

```python
def cognitive_model1(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid Model-Based/Model-Free learner where the mixing weight 'w' is modulated by OCI.
    
    This tests the hypothesis that OCI symptoms shift the balance between 
    goal-directed (MB) and habitual (MF) control systems.
    
    Parameters:
    learning_rate: [0, 1] Learning rate for value updates.
    beta: [0, 10] Inverse temperature for softmax choice.
    w_base: [0, 1] Baseline weight for Model-Based control (0=Pure MF, 1=Pure MB).
    w_oci_mod: [-1, 1] Modulation of w by OCI score.
    """
    learning_rate, beta, w_base, w_oci_mod = model_parameters
    n_trials = len(action_1)
    oci_score = oci[0]
    
    # Calculate effective w based on OCI, constrained to [0, 1]
    w = w_base + (w_oci_mod * oci_score)
    w = np.clip(w, 0.0, 1.0)

    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Initialize Q-values
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))

    for trial in range(n_trials):
        # --- Stage 1 Choice ---
        # Model-Based value calculation: Bellman equation using transition matrix
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Integrated value: weighted sum of MB and MF values
        q_net = (w * q_stage1_mb) + ((1 - w) * q_stage1_mf)
        
        # Softmax policy for stage 1
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        state_idx = int(state[trial])
        a1 = int(action_1[trial])

        # --- Stage 2 Choice ---
        # Standard Model-Free Q-learning for the second stage
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
        
        a2 = int(action_2[trial])
        r = reward[trial]

        # --- Learning Updates ---
        # Stage 2 TD Error
        delta_stage2 = r - q_stage2_mf[state_idx, a2]
        
        # Stage 1 TD Error (SARSA-style for MF update)
        delta_stage1 = q_stage2_mf[state_idx, a2] - q_stage1_mf[a1]
        
        # Update Q-values
        q_stage2_mf[state_idx, a2] += learning_rate * delta_stage2
        q_stage1_mf[a1] += learning_rate * delta_stage1
        
        # Eligibility trace: Stage 1 MF values also learn from Stage 2 reward outcome
        q_stage1_mf[a1] += learning_rate * delta_stage2 # Lambda=1 assumed for simplicity

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: Asymmetric Learning Rates Modulated by OCI
This model investigates if OCI relates to a bias in learning from positive versus negative feedback. It proposes that the learning rate for negative prediction errors (`alpha_neg`) is distinct from positive ones (`alpha_pos`), and that OCI specifically exacerbates or diminishes the sensitivity to negative outcomes (punishment/loss).

```python
def cognitive_model2(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Free learner with asymmetric learning rates for positive/negative prediction errors,
    where the negative learning rate is modulated by OCI.
    
    This tests if OCI is associated with hyper-sensitivity or hypo-sensitivity 
    to negative feedback (loss avoidance).
    
    Parameters:
    alpha_pos: [0, 1] Learning rate for positive prediction errors.
    alpha_neg_base: [0, 1] Base learning rate for negative prediction errors.
    alpha_neg_oci_slope: [-1, 1] Modulation of negative learning rate by OCI.
    beta: [0, 10] Inverse temperature.
    """
    alpha_pos, alpha_neg_base, alpha_neg_oci_slope, beta = model_parameters
    n_trials = len(action_1)
    oci_score = oci[0]
    
    # Calculate effective negative learning rate
    alpha_neg = alpha_neg_base + (alpha_neg_oci_slope * oci_score)
    alpha_neg = np.clip(alpha_neg, 0.0, 1.0)

    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))

    for trial in range(n_trials):
        # --- Stage 1 Choice ---
        exp_q1 = np.exp(beta * q_stage1_mf)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        state_idx = int(state[trial])
        a1 = int(action_1[trial])

        # --- Stage 2 Choice ---
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
        
        a2 = int(action_2[trial])
        r = reward[trial]

        # --- Learning Updates ---
        
        # Stage 2 Update
        delta_stage2 = r - q_stage2_mf[state_idx, a2]
        lr_stage2 = alpha_pos if delta_stage2 > 0 else alpha_neg
        q_stage2_mf[state_idx, a2] += lr_stage2 * delta_stage2
        
        # Stage 1 Update (TD-0)
        # Using the updated stage 2 value to drive stage 1 update (common variant)
        # or standard TD error using pre-update values. Here we use standard.
        delta_stage1 = q_stage2_mf[state_idx, a2] - q_stage1_mf[a1]
        
        # Determine learning rate based on the sign of the prediction error
        # Note: The effective error for Stage 1 often includes the Stage 2 outcome in full reinforcement
        # Here we apply the asymmetry to the direct TD update.
        lr_stage1 = alpha_pos if delta_stage1 > 0 else alpha_neg
        q_stage1_mf[a1] += lr_stage1 * delta_stage1
        
        # Direct reinforcement of Stage 1 from Reward (Eligibility Trace logic)
        # We apply the asymmetry to this direct reward signal as well
        lr_direct = alpha_pos if r > 0 else alpha_neg # Simple approximation based on reward sign
        q_stage1_mf[a1] += lr_direct * delta_stage2 

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: OCI-Modulated Stickiness (Choice Perseveration)
This model posits that OCI relates to behavioral rigidity or "stickiness"â€”the tendency to repeat the previous action regardless of reward outcome. It modifies a standard Hybrid learner by adding a stickiness parameter that is a function of the OCI score. This separates the concept of "learning" (value update) from "repetition" (motor/choice history).

```python
def cognitive_model3(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid Model with OCI-modulated Stickiness (Perseveration).
    
    This tests if OCI is associated with a raw tendency to repeat choices 
    (compulsivity/habit) independent of value-based learning.
    
    Parameters:
    learning_rate: [0, 1] Learning rate.
    beta: [0, 10] Inverse temperature.
    w: [0, 1] Mixing weight (0=MF, 1=MB).
    stickiness_base: [0, 5] Base bonus added to previously chosen action Q-value.
    stickiness_oci_mod: [-2, 2] Modulation of stickiness by OCI.
    """
    learning_rate, beta, w, stickiness_base, stickiness_oci_mod = model_parameters
    n_trials = len(action_1)
    oci_score = oci[0]
    
    # Calculate effective stickiness
    stickiness = stickiness_base + (stickiness_oci_mod * oci_score)
    # Stickiness can be negative (alternation) or positive (repetition), 
    # but usually we bound magnitude reasonable for Q-values (~0-1 range rewards)
    
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))
    
    prev_action_1 = -1

    for trial in range(n_trials):
        # --- Stage 1 Choice ---
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        q_net = (w * q_stage1_mb) + ((1 - w) * q_stage1_mf)
        
        # Add stickiness bonus to the net Q-value of the previous action
        if prev_action_1 != -1:
            q_net[prev_action_1] += stickiness
            
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[int(action_1[trial])]
        
        state_idx = int(state[trial])
        a1 = int(action_1[trial])
        prev_action_1 = a1 # Update previous action for next trial

        # --- Stage 2 Choice ---
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[int(action_2[trial])]
        
        a2 = int(action_2[trial])
        r = reward[trial]

        # --- Learning Updates ---
        delta_stage2 = r - q_stage2_mf[state_idx, a2]
        delta_stage1 = q_stage2_mf[state_idx, a2] - q_stage1_mf[a1]
        
        q_stage2_mf[state_idx, a2] += learning_rate * delta_stage2
        q_stage1_mf[a1] += learning_rate * delta_stage1
        q_stage1_mf[a1] += learning_rate * delta_stage2 # Eligibility trace

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
