Here are three new cognitive models based on the two-step task and the participant's OCI score. These models explore different mechanisms of how obsessive-compulsive traits might influence reinforcement learning, specifically focusing on perseveration, model-based control balance, and learning rate modulation.

### Model 1: Hybrid RL with OCI-Driven Perseveration (Sticky Choice)
This model hypothesizes that OCI symptoms manifest as "stickiness" or perseverationâ€”a tendency to repeat previous choices regardless of reward outcome. The degree of this stickiness is modulated by the OCI score. It uses a hybrid Model-Based/Model-Free architecture where the mixing weight `w` is fixed, but the choice stickiness is variable.

```python
def cognitive_model1(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid RL with OCI-modulated choice stickiness (perseveration).
    
    Hypothesis: Higher OCI scores lead to higher choice perseveration (stickiness),
    making it harder to switch away from previous actions regardless of outcomes.
    
    Parameters:
    - learning_rate: [0, 1] Learning rate for value updates.
    - beta: [0, 10] Inverse temperature for softmax.
    - w: [0, 1] Mixing weight (0=Model-Free, 1=Model-Based).
    - stick_base: [0, 5] Base level of choice stickiness.
    - stick_oci_mod: [-5, 5] Modulation of stickiness by OCI score.
    """
    learning_rate, beta, w, stick_base, stick_oci_mod = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
    
    # Calculate effective stickiness based on OCI
    stickiness = stick_base + (stick_oci_mod * current_oci)
    
    # Initialize Q-values
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Track previous choice for stickiness
    prev_a1 = -1
    
    for trial in range(n_trials):
        a1 = int(action_1[trial])
        s_idx = int(state[trial])
        a2 = int(action_2[trial])
        r = reward[trial]

        # Handle missing data
        if a1 == -1 or s_idx == -1 or a2 == -1:
            p_choice_1[trial] = 0.5
            p_choice_2[trial] = 0.5
            continue

        # --- Stage 1 Choice ---
        # Model-Based Value
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Combined Value
        q_net = (w * q_stage1_mb) + ((1 - w) * q_stage1_mf)
        
        # Add stickiness bonus to the previously chosen action
        stick_vector = np.zeros(2)
        if prev_a1 != -1:
            stick_vector[prev_a1] = stickiness
            
        # Softmax
        exp_q1 = np.exp(beta * q_net + stick_vector)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[a1]
        
        # Update stickiness tracker
        prev_a1 = a1

        # --- Stage 2 Choice ---
        exp_q2 = np.exp(beta * q_stage2_mf[s_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[a2]

        # --- Learning Updates ---
        # Stage 2 Update (TD)
        delta2 = r - q_stage2_mf[s_idx, a2]
        q_stage2_mf[s_idx, a2] += learning_rate * delta2
        
        # Stage 1 Update (TD)
        delta1 = q_stage2_mf[s_idx, a2] - q_stage1_mf[a1]
        q_stage1_mf[a1] += learning_rate * delta1

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: OCI-Modulated Model-Based/Model-Free Trade-off with Decay
This model posits that OCI affects the balance between Model-Based (goal-directed) and Model-Free (habitual) control. Specifically, high OCI might be associated with a stronger reliance on rigid habitual (Model-Free) systems or, conversely, an over-reliance on Model-Based planning. The mixing weight `w` is a logistic function of the OCI score. Additionally, it includes a decay parameter for unchosen options, reflecting memory constraints.

```python
def cognitive_model2(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid RL where OCI determines the Model-Based vs. Model-Free balance (w).
    
    Hypothesis: OCI score shifts the strategy between planning (MB) and habit (MF).
    This uses a logistic function to bound w between 0 and 1 based on OCI.
    
    Parameters:
    - learning_rate: [0, 1] Learning rate.
    - beta: [0, 10] Inverse temperature.
    - w_intercept: [-5, 5] Base log-odds of using Model-Based control.
    - w_oci_slope: [-5, 5] How strongly OCI affects the MB/MF balance.
    - decay: [0, 1] Forgetting rate for unvisited states/actions.
    """
    learning_rate, beta, w_intercept, w_oci_slope, decay = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
    
    # Calculate w using logistic function to keep it in [0,1]
    # w = 1 / (1 + exp(-(intercept + slope * oci)))
    logit_w = w_intercept + (w_oci_slope * current_oci)
    w = 1.0 / (1.0 + np.exp(-logit_w))
    
    q_stage1_mf = np.zeros(2)
    q_stage2_mf = np.zeros((2, 2))
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    for trial in range(n_trials):
        a1 = int(action_1[trial])
        s_idx = int(state[trial])
        a2 = int(action_2[trial])
        r = reward[trial]

        if a1 == -1 or s_idx == -1 or a2 == -1:
            p_choice_1[trial] = 0.5
            p_choice_2[trial] = 0.5
            continue

        # --- Stage 1 Choice ---
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Mix MB and MF
        q_net = (w * q_stage1_mb) + ((1 - w) * q_stage1_mf)
        
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[a1]

        # --- Stage 2 Choice ---
        exp_q2 = np.exp(beta * q_stage2_mf[s_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[a2]

        # --- Learning Updates with Decay ---
        
        # Stage 2 Update
        delta2 = r - q_stage2_mf[s_idx, a2]
        q_stage2_mf[s_idx, a2] += learning_rate * delta2
        # Decay unchosen in current state
        q_stage2_mf[s_idx, 1-a2] *= (1 - decay)
        # Decay other state entirely
        q_stage2_mf[1-s_idx, :] *= (1 - decay)

        # Stage 1 Update
        delta1 = q_stage2_mf[s_idx, a2] - q_stage1_mf[a1]
        q_stage1_mf[a1] += learning_rate * delta1
        # Decay unchosen stage 1
        q_stage1_mf[1-a1] *= (1 - decay)

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: Separate Learning Rates for Wins and Losses Modulated by OCI
This model explores the idea that OCI is related to altered sensitivity to punishment (losses/omissions) versus reward. It implements a simple Model-Free system (often sufficient if the participant is very habitual) but splits the learning rate into `alpha_win` and `alpha_loss`. The `alpha_loss` is specifically modulated by the OCI score, testing the hypothesis that OCI increases learning from negative outcomes.

```python
def cognitive_model3(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Free RL with split learning rates, where OCI modulates Loss Learning Rate.
    
    Hypothesis: OCI specifically impacts how much weight is given to negative outcomes
    (learning from losses/omissions), while reward learning remains baseline.
    
    Parameters:
    - lr_win: [0, 1] Learning rate for positive rewards (1.0).
    - lr_loss_base: [0, 1] Base learning rate for zero rewards (0.0).
    - lr_loss_oci_slope: [-1, 1] OCI modulation of loss learning rate.
    - beta: [0, 10] Inverse temperature.
    - eligibility: [0, 1] Eligibility trace (how much Stage 2 outcome affects Stage 1).
    """
    lr_win, lr_loss_base, lr_loss_oci_slope, beta, eligibility = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0]
    
    # Calculate effective loss learning rate (bounded [0,1])
    # If slope is positive, higher OCI -> faster learning from failure
    effective_lr_loss = lr_loss_base + (lr_loss_oci_slope * current_oci)
    effective_lr_loss = np.clip(effective_lr_loss, 0.0, 1.0)
    
    q_stage1 = np.zeros(2)
    q_stage2 = np.zeros((2, 2))
    
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    for trial in range(n_trials):
        a1 = int(action_1[trial])
        s_idx = int(state[trial])
        a2 = int(action_2[trial])
        r = reward[trial]

        if a1 == -1 or s_idx == -1 or a2 == -1:
            p_choice_1[trial] = 0.5
            p_choice_2[trial] = 0.5
            continue
            
        # Determine current learning rate based on outcome
        current_lr = lr_win if r > 0 else effective_lr_loss

        # --- Stage 1 Choice ---
        exp_q1 = np.exp(beta * q_stage1)
        probs_1 = exp_q1 / np.sum(exp_q1)
        p_choice_1[trial] = probs_1[a1]

        # --- Stage 2 Choice ---
        exp_q2 = np.exp(beta * q_stage2[s_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        p_choice_2[trial] = probs_2[a2]

        # --- Updates ---
        # Stage 2 TD Error
        delta2 = r - q_stage2[s_idx, a2]
        q_stage2[s_idx, a2] += current_lr * delta2
        
        # Stage 1 TD Error (SARSA-like update using Stage 2 Q-value)
        # Note: In pure MF, stage 1 is updated by stage 2 value
        delta1 = q_stage2[s_idx, a2] - q_stage1[a1]
        
        # Eligibility trace allows the Stage 2 outcome (r) to directly impact Stage 1 Q
        # Standard TD(1) approximation for two-step
        combined_delta = delta1 + (eligibility * delta2)
        q_stage1[a1] += current_lr * combined_delta

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
