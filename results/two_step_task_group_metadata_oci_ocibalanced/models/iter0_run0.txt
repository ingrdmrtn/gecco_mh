Here are three cognitive models that explain the participant data, incorporating the OCI score to modulate learning and decision-making processes.

### Model 1: Hybrid Learner with OCI-modulated Model-Based Control
This model assumes participants use a mixture of Model-Free (MF) and Model-Based (MB) reinforcement learning. The `w` parameter controls the balance between these two systems. Crucially, the OCI score modulates this balance, hypothesizing that higher OCI scores might lead to a more rigid, habit-based (MF) strategy or an over-reliance on explicit structure (MB), depending on the specific cognitive theory being tested. Here, we model higher OCI as biasing towards Model-Free control (habitual behavior).

```python
def cognitive_model1(action_1, state, action_2, reward, oci, model_parameters):
    """
    Hybrid RL model where the balance between Model-Based and Model-Free control 
    is modulated by the OCI score.
    
    Hypothesis: Higher OCI scores reduce the weight of Model-Based planning (w), 
    leading to more Model-Free (habitual) behavior.
    
    Parameters:
    learning_rate: [0,1] Rate at which Q-values are updated.
    beta: [0,10] Inverse temperature for softmax choice.
    w_base: [0,1] Baseline weighting for Model-Based control (0=Pure MF, 1=Pure MB).
    oci_sens: [0,1] Sensitivity of the weighting parameter 'w' to the OCI score.
    """
    learning_rate, beta, w_base, oci_sens = model_parameters
    n_trials = len(action_1)
    # Normalize OCI to be roughly 0-1 range for stability if raw score is high, 
    # but here we assume the input is scalable. We take the first element as it's group metadata.
    current_oci = oci[0] / 20.0 # Scaling heuristic assuming OCI range 0-20 approx for stability
    
    # Calculate effective w. We clip to ensure it stays in [0,1].
    # Higher OCI reduces w, pushing towards MF.
    w = w_base - (oci_sens * current_oci)
    w = np.clip(w, 0.0, 1.0)
  
    transition_matrix = np.array([[0.7, 0.3], [0.3, 0.7]])
    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    
    # Q-values
    q_stage1_mf = np.zeros(2) # Model-free values for stage 1
    q_stage2_mf = np.zeros((2, 2)) # Values for stage 2 (2 states, 2 actions)

    for trial in range(n_trials):
        # --- Stage 1 Policy ---
        # Model-Based valuation: Q_MB = T * max(Q_stage2)
        max_q_stage2 = np.max(q_stage2_mf, axis=1)
        q_stage1_mb = transition_matrix @ max_q_stage2
        
        # Hybrid valuation: Q_net = w * Q_MB + (1-w) * Q_MF
        q_net = w * q_stage1_mb + (1 - w) * q_stage1_mf
        
        exp_q1 = np.exp(beta * q_net)
        probs_1 = exp_q1 / np.sum(exp_q1)
        
        # Handle missing data or invalid actions gracefully if present (though usually cleaned)
        a1 = int(action_1[trial])
        p_choice_1[trial] = probs_1[a1]
        
        state_idx = int(state[trial])

        # --- Stage 2 Policy ---
        # Standard softmax on stage 2 Q-values
        exp_q2 = np.exp(beta * q_stage2_mf[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        
        a2 = int(action_2[trial])
        # Check for valid action (handling the -1 case in participant 3)
        if a2 == -1:
            p_choice_2[trial] = 1.0 # No choice made, ignore likelihood contribution
            r = 0 # No reward
        else:
            p_choice_2[trial] = probs_2[a2]
            r = reward[trial]
            
            # --- Updating ---
            # SARSA / TD(1) style updates
            
            # Stage 2 update (Reward Prediction Error)
            delta_stage2 = r - q_stage2_mf[state_idx, a2]
            q_stage2_mf[state_idx, a2] += learning_rate * delta_stage2
            
            # Stage 1 update (Model-Free)
            # TD(1) update: Stage 1 value driven by Stage 2 RPE
            q_stage1_mf[a1] += learning_rate * delta_stage2
            
            # Note: We could also do TD(0) using the stage 2 value, but TD(1) 
            # using the final reward is common in this task paradigm.

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 2: Perseveration Model with OCI-modulated Stickiness
This model focuses on "stickiness" or perseverationâ€”the tendency to repeat the previous choice regardless of reward. Here, the OCI score modulates the `stickiness` parameter. The hypothesis is that individuals with higher compulsive traits (higher OCI) are more likely to repetitively choose the same spaceship (Stage 1 action), regardless of whether it was rewarded, reflecting a compulsion to repeat.

```python
def cognitive_model2(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Free learner with OCI-modulated choice perseveration (stickiness).
    
    Hypothesis: Higher OCI scores increase the 'stickiness' parameter, making
    participants more likely to repeat their previous stage 1 choice regardless of outcome.
    
    Parameters:
    learning_rate: [0,1] Value update rate.
    beta: [0,10] Inverse temperature.
    stickiness_base: [0,5] Baseline bonus added to the Q-value of the previously chosen action.
    oci_sticky_boost: [0,1] How much OCI increases the stickiness factor.
    """
    learning_rate, beta, stickiness_base, oci_sticky_boost = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0] / 20.0
    
    # Effective stickiness increases with OCI
    eff_stickiness = stickiness_base + (oci_sticky_boost * current_oci * 5.0) 

    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    q_stage1 = np.zeros(2)
    q_stage2 = np.zeros((2, 2))
    
    prev_action_1 = -1

    for trial in range(n_trials):
        # --- Stage 1 Policy ---
        # Add stickiness bonus to the Q-value of the previous action
        q_stage1_modified = q_stage1.copy()
        if prev_action_1 != -1:
            q_stage1_modified[prev_action_1] += eff_stickiness

        exp_q1 = np.exp(beta * q_stage1_modified)
        probs_1 = exp_q1 / np.sum(exp_q1)
        
        a1 = int(action_1[trial])
        p_choice_1[trial] = probs_1[a1]
        
        state_idx = int(state[trial])

        # --- Stage 2 Policy ---
        exp_q2 = np.exp(beta * q_stage2[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        
        a2 = int(action_2[trial])
        
        if a2 == -1:
            p_choice_2[trial] = 1.0
            r = 0
        else:
            p_choice_2[trial] = probs_2[a2]
            r = reward[trial]

            # --- Updating ---
            # Standard Model-Free Q-learning (TD(1) logic for Stage 1)
            delta_stage2 = r - q_stage2[state_idx, a2]
            q_stage2[state_idx, a2] += learning_rate * delta_stage2
            
            # Update Stage 1 based on final outcome
            q_stage1[a1] += learning_rate * delta_stage2
        
        # Store action for next trial's stickiness
        prev_action_1 = a1

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```

### Model 3: Separate Learning Rates for Positive/Negative Feedback
This model investigates asymmetry in learning. It proposes that participants might learn differently from rewards (gold coins) versus non-rewards (0 coins). The OCI score is used to modulate the learning rate specifically for negative outcomes (or lack of reward), hypothesizing that high OCI individuals might be hyper-sensitive (or hypo-sensitive) to "failure" or missed rewards.

```python
def cognitive_model3(action_1, state, action_2, reward, oci, model_parameters):
    """
    Model-Free learner with separate learning rates for positive (alpha_pos) 
    and negative/zero (alpha_neg) outcomes. OCI modulates alpha_neg.
    
    Hypothesis: OCI affects how strongly participants update their beliefs 
    after receiving 0 coins (disappointment/error sensitivity).
    
    Parameters:
    alpha_pos: [0,1] Learning rate for rewarded trials (reward=1).
    alpha_neg_base: [0,1] Base learning rate for unrewarded trials (reward=0).
    beta: [0,10] Inverse temperature.
    oci_neg_mod: [0,1] Modulation of negative learning rate by OCI. 
                   (Higher OCI -> Higher learning from failure).
    """
    alpha_pos, alpha_neg_base, beta, oci_neg_mod = model_parameters
    n_trials = len(action_1)
    current_oci = oci[0] / 20.0
    
    # Calculate effective negative learning rate
    # We assume OCI increases sensitivity to negative feedback
    alpha_neg = alpha_neg_base + (oci_neg_mod * current_oci)
    alpha_neg = np.clip(alpha_neg, 0.0, 1.0)

    p_choice_1 = np.zeros(n_trials)
    p_choice_2 = np.zeros(n_trials)
    q_stage1 = np.zeros(2)
    q_stage2 = np.zeros((2, 2))

    for trial in range(n_trials):
        # --- Stage 1 Policy ---
        exp_q1 = np.exp(beta * q_stage1)
        probs_1 = exp_q1 / np.sum(exp_q1)
        
        a1 = int(action_1[trial])
        p_choice_1[trial] = probs_1[a1]
        
        state_idx = int(state[trial])

        # --- Stage 2 Policy ---
        exp_q2 = np.exp(beta * q_stage2[state_idx])
        probs_2 = exp_q2 / np.sum(exp_q2)
        
        a2 = int(action_2[trial])
        
        if a2 == -1:
            p_choice_2[trial] = 1.0
            r = 0
            current_alpha = alpha_neg # Treat missing as unrewarded/neutral
        else:
            p_choice_2[trial] = probs_2[a2]
            r = reward[trial]
            
            # Select learning rate based on outcome
            if r > 0:
                current_alpha = alpha_pos
            else:
                current_alpha = alpha_neg

            # --- Updating ---
            # Update Stage 2
            delta_stage2 = r - q_stage2[state_idx, a2]
            q_stage2[state_idx, a2] += current_alpha * delta_stage2
            
            # Update Stage 1 (Model-Free TD(1))
            q_stage1[a1] += current_alpha * delta_stage2

    eps = 1e-10
    log_loss = -(np.sum(np.log(p_choice_1 + eps)) + np.sum(np.log(p_choice_2 + eps)))
    return log_loss
```
